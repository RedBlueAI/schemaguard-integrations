# SchemaGuard Integrations

API integration adapters for third-party services. These adapters handle schema mapping between our internal data models and external API contracts.

When a third-party API changes its schema, SchemaGuard AI automatically detects the drift, generates corrective adapter code, and submits a pull request to this repository.

## Structure

```
src/
  adapters/
    inventory.ts      - Inventory Management API adapter
    workorders.ts     - Work Order API adapter
    billing.ts        - Billing & Invoicing API adapter
    iot.ts            - IoT Sensor Data API adapter
    compliance.ts     - Compliance & Safety API adapter
  types/
    inventory.ts      - Inventory API types
    workorders.ts     - Work Order API types
    billing.ts        - Billing API types
    iot.ts            - IoT API types
    compliance.ts     - Compliance API types
  tests/
    inventory.test.ts - Inventory adapter tests
    workorders.test.ts
    billing.test.ts
    iot.test.ts
    compliance.test.ts
```

## Auto-Generated PRs

Pull requests in this repository are automatically generated by the SchemaGuard AI agent pipeline when schema drift is detected. Each PR includes:
- Schema change summary
- Generated corrective code
- Test results
- Ticket reference and SLA information
